## What was fixed
1. **Theme callback normalization**
   - `ThemeProvider` now resolves requested theme IDs before applying + persisting + emitting callbacks.
   - Prevents callback leakage of unresolved/system IDs like `'light'` when custom themes use different IDs.
   - File: `components/themes/theme-provider.tsx`

2. **Provider regression lock-in**
   - Added a unit test asserting `onThemeChange` receives normalized theme IDs.
   - Updated expectations from hard-coded aliases (`tron-dark`) to configured fallback (`black-white`).
   - File: `tests/unit/theme-provider.test.tsx`

3. **Integration robustness**
   - Rapid-switching assertion now checks for terminal valid IDs instead of a fixed alias.
   - File: `tests/integration/theme-switching.test.tsx`

4. **`setTheme` dedupe fix**
   - Adjusted no-op suppression to ignore redundant updates only for already-active *known* themes while still allowing normalized fallback IDs to persist/callback.
   - File: `components/themes/theme-provider.tsx`

5. **Legacy stack cleanup completed**
